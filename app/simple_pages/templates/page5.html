Definitions


Instantiation:

Creating an object of a given class.


Object:

A self-contained component that consists of both methods and properties to utilize data.


Class:

A blueprint for creating objects, providing initial values for their state, and implementations of behavior.


Namespace:

A region that provides a scope to the identifiers inside of it, used to prevent name collisions.


Constructor:

A subroutine called to create an object, it prepares the object for use and often accepts arguments used for setting member variables.


Fixture:

The understood and fixed environment in which tests are run to get consistent results, our docker setup is part of our fixture.


Type Hint:

A way to statically indicate type of values in python, introduced in Python 3.5.


Type Cast:

Methods of changing an expression from one data type to another, such as int to float.


Unit Test:

A way of testing a unit, the smallest piece of code that can be reasonably isolated in a system.


Static:

Unchanging. Static members of a class are only created once, regardless of how many objects of the class are created.


Instance Method:

A method that belongs to the instance of a class, rather than the class itself. Non-static.


Instance Property:

A property that belongs to the instance of the class, rather than the class itself.


Static Method:

Static methods belong to the class rather than the instance of the class.


Static Property:

A property that belongs to the class rather than the instance of the class.


Encapsulation:

Bundling of data with the methods that operate on it, a mechanism of restricting access.


Inheritance:

Where one class inherits the attributes and methods of another class, with a parent-child relationship.


Polymorphism:

Similar to inheritance, but where one interface is implemented by multiple different classes.


Abstraction:

Hiding all but the relevant data about an object to reduce complexity, increase efficiency, decouple design from implementation.


Overview of SOLID:

Hiding all but the relevant data about an object to reduce complexity, increase efficiency, decouple design from implementation.


Design Patterns:

A general, reusable solution to a commonly occuring problem within a context in software design. It cannot be transformed directly into source or machine code, as it is an abstraction of codes, and demonstrates principles of solving this problem.
