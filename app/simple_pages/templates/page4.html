Github Actions:

Github actions automatically perform tasks upon pushes, specifically it can be used to immediately upload to Heroku. My project has two workflows to upload to development and upload to production respectively.


Development Server:

A development server allows a programmer to make frequent changes without risking the integrity of the production output. It also allows changes to be rolled back on the development server far more cleanly than having to rollback presented code.


Deploy Master:

Deploying master in this case automatically deploy it to production due to the prod.yml workflow and github actions that are set. Alternatively, you could set each branch to be tracked by Heroku manually, and deploy development and production branches like that.


Image Deployment:

Briefly, due to github having secrets corresponding to my dockerhub login and password, and due to having a portion in workflow dedicated to logging into dockerhub and deploying my production code, image deployment occurs automatically.
