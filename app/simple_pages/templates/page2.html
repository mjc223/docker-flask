Containerization:

A container is essentially a package of software code with only the bare minimum OS libraries required to make the program function. The container can run universally on all hosts, due to the abstraction that removes the host operating system from the equation. Containerization in comparison to virtualization runs with far less overhead and far greater simplicity. Docker and the Docker Engine is a tool to enable containerization on hardware.


Images:

An image in the context of computing is essentially a saved state of something, whether it is a disk image or a docker image. It is a read only file that contains all the relevant information about a particular storage device or system state. In docker, an image is vital to containerization. An image must be built and utilized in order to construct a container. Commands:

A brief summary of vital docker commands.
docker compose up: Utilizes a user created YAML file to run an app in docker
docker create : Creates new container
docker images : Lists all local docker images in the repository
docker ps : Lists all currently running docker containers, append --all for all created ones
docker stop docker restart : Stop or restart a specific docker container
docker rm : Remove a specific docker container
